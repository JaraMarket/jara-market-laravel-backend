# Laravel MCP Manager Skill

This skill guides the creation and integration of new Model Context Protocol (MCP) tools into the Laravel backend using the `laravel/mcp` package.

## 🛠️ Tool Creation Workflow

### 1. Identify the Need
Determine the capability needed (e.g., creating products, managing orders, database backups).

### 2. Generate the Tool Class
Use `php artisan make:class Mcp/Tools/YourNewTool` or create the file manually in `app/Mcp/Tools/`.

**Template:**
```php
<?php

namespace App\Mcp\Tools;

use Illuminate\Contracts\JsonSchema\JsonSchema;
use Laravel\Mcp\Request;
use Laravel\Mcp\Response;
use Laravel\Mcp\Server\Attributes\Description;
use Laravel\Mcp\Server\Tool;

#[Description('Briefly describe what this tool does.')]
class YourNewTool extends Tool
{
    public function handle(Request $request): Response
    {
        // 1. Extract arguments
        // 2. Perform logic (e.g., Product::create([...]))
        // 3. Return Response::text() or Response::json()
    }

    public function schema(JsonSchema $schema): array
    {
        return [
            'param_name' => $schema->string()->description('Param description'),
        ];
    }
}
```

### 3. Register the Tool
Add the tool class to the `$tools` array in `app/Mcp/Servers/JaraMarketServer.php`.

```php
protected array $tools = [
    AuditProductsTool::class,
    YourNewTool::class, // Add it here
];
```

## 🚫 Critical Rules (Persistence)
All tools MUST abide by the rules in `AGENT_RULES.md`:
*   **No Destructive Actions:** Tools must not clear tables unless explicitly requested for maintenance.
*   **Backup First:** Any tool performing bulk updates should ideally trigger a `mysqldump` backup before starting.

## 🚀 Recommended Tool Roadmap
1.  **`PopulateProductsTool`**: To handle the 2,000+ entries.
2.  **`OrderManagementTool`**: To update order statuses.
3.  **`InventoryUpdateTool`**: To manage stock levels.
4.  **`DatabaseBackupTool`**: To automate SQL snapshots.

# Data Population Strategy Skill

This skill provides a structured approach to migrating or populating large volumes of data into the JaraMarket backend.

## 📋 Pre-Population Checklist
1.  **Source Data Analysis**: Identify the source format (e.g., a list of 2,000 products).
2.  **Model Mapping**: Map source fields to `Product` model attributes (name, price, stock, category_id).
3.  **Validation**: Ensure all required fields are present to avoid database constraint errors.
4.  **Backup**: Run `mysqldump` to create a "point-in-time" restore point before starting.

## 🚀 Execution Workflow

### 1. The Chunking Strategy
Do not attempt to import 2,000 items in one single request. Break the data into **chunks of 50-100 items** to prevent timeouts and memory issues.

### 2. Using the MCP Tools
Use the `laravel-mcp-manager` skill to ensure the `PopulateProductsTool` is ready and registered.

### 3. Verification & Cleanup
*   Run the `AuditProductsTool` after every 5 chunks to verify data integrity.
*   Log any failed entries to a temporary file for manual review.

## 🔒 Post-Population "Freeze"
Once all 2,000 entries are verified:
1.  **Export SQL**: Generate a full backup of the `products`, `categories`, and `ingredients` tables.
2.  **Generate Seeder**: Create a `ProductionDataSeeder.php` that contains the verified data.

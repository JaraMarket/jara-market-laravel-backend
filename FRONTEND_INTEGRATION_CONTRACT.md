# 📜 JaraMarket: Frontend Integration Contract (Production)

This document is the definitive guide for integrating the JaraMarket Flutter applications (Customer & Vendor) with the Laravel backend. **No external searching or Swagger login is required.**

---

## 🏗️ Executive Summary
*   **Base URL:** `https://jara-market-laravel-backend-production.up.railway.app/api/jaram`
*   **Auth Strategy:** Laravel Sanctum (Bearer Token)
*   **Response Format:** Always JSON.
*   **Standard Response Object:**
    ```json
    {
        "status": true,
        "message": "Operation successful",
        "data": { ... }
    }
    ```

## 🔑 1. Authentication Flow
All protected requests must include the header:
`Authorization: Bearer <token>`

### 1.1 Login
*   **Endpoint:** `POST /login`
*   **Payload:**
    ```json
    {
        "email": "user@example.com",
        "password": "yourpassword"
    }
    ```
*   **Response:** Returns a `token` string and a `user` object. Store this token securely.

### 1.2 Registration
*   **Endpoint:** `POST /register`
*   **Payload:**
    ```json
    {
        "firstname": "John",
        "lastname": "Doe",
        "email": "john@example.com",
        "password": "password123",
        "phone_number": "08012345678",
        "role": "customer",
        "referral_code": "optional_code"
    }
    ```
*   **Roles:** `customer`, `vendor`.

### 1.3 Fetch User Profile
*   **Endpoint:** `GET /fetch-user` (Auth Required)
*   **Purpose:** Use this to verify if a token is still valid on app launch.

## 🥗 2. Catalog & Products

### 2.1 Fetch All Products
*   **Endpoint:** `GET /fetch/product` (Auth Required)
*   **Description:** Returns the full catalog of meals available for purchase.

### 2.2 Fetch Ingredients
*   **Endpoint:** `GET /fetch/ingredients` (Auth Required)
*   **Description:** Returns individual raw materials/ingredients.

### 2.3 Fetch Categories
*   **Endpoint:** `GET /vendors/categories` (Public)
*   **Description:** Returns categories like "Rice", "Soups", "Drinks".

## 🛒 3. Order Management

### 3.1 Create Order
*   **Endpoint:** `POST /orders` (Auth Required)
*   **Payload Template:**
    ```json
    {
        "order_date": "2026-05-10",
        "delivery_type": "delivery", 
        "total": 5000.00,
        "service_charge": 500.00,
        "address_id": 1,
        "products": [
            { "product_id": 1, "quantity": 2, "price": 1500.00 }
        ],
        "ingredients": [
            { "ingredient_id": 5, "quantity": 1, "price": 2000.00, "unit": "kg" }
        ]
    }
    ```
*   **Delivery Types:** `pickup`, `delivery`, `walkin`.

## 💰 4. Wallet & Payments

### 4.1 Check Wallet Balance
*   **Endpoint:** `GET /wallet` (Auth Required)
*   **Response:**
    ```json
    {
        "status": true,
        "data": {
            "balance": "10,500.00",
            "currency": "NGN"
        }
    }
    ```

### 4.2 Initialize Payment (Paystack)
*   **Endpoint:** `POST /payments/initialize-transaction` (Auth Required)
*   **Payload:**
    ```json
    {
        "amount": 5000,
        "email": "user@example.com"
    }
    ```
*   **Response:** Returns a `checkout_url`. Redirect the user to this URL for payment.

## 🛠️ Developer Checklist
1.  **Headers:** Always set `Accept: application/json` and `Content-Type: application/json`.
2.  **Base URL:** Set `https://jara-market-laravel-backend-production.up.railway.app/api/jaram` as your global base.
3.  **Authentication:** Never hardcode the Master Key in production apps. Use the `/login` flow.
4.  **Security:** Use `flutter_secure_storage` for storing Bearer tokens.
